02 · Issues Found

What was breaking on mobile

Click any issue to expand and see the technical detail.

1. Form inputs caused iOS auto-zoom

Critical

Form fields used font-size:15px. On iPhone Safari, any input under 16px triggers an auto-zoom when tapped — the browser zooms in to make the text readable. The user then has to manually pinch-zoom back out. This is the single biggest mobile UX killer on form-heavy sites.

Before.field input { font-size: 15px; color: #fff; }
Afterinput, select, textarea { font-size: 16px !important; color: #2d2418 !important; }

2. Form text was invisible on cream theme

Critical

The base form CSS still had color:#fff left over from the old dark theme. Only the explicit .quote-form override fixed it. Any other form on the site (contact, future newsletter signup, etc.) would render WHITE TEXT on CREAM BACKGROUND — totally invisible.

Beforecolor: #fff (invisible on cream)
Aftercolor: #2d2418 (warm dark, readable)

3. Comparison table overflowed the viewport

Critical

The "Party Bus R Us vs. Competitors" table on the home page is wider than a phone screen. Without a scroll wrapper, it forced a horizontal page scroll, which on iOS causes the whole page to feel "broken" — users swipe and the layout shifts unexpectedly.

Before<table> with no scroll wrapper → page overflows
Afterdisplay:block; overflow-x:auto on small screens. Table scrolls inside its own container.

4. Hamburger menu button was too small to tap

Important

The .menu-btn had no min-width or min-height — just a 24px font size in a transparent button. Apple's HIG and WCAG 2.1 both require 44×44px minimum for tap targets. Anything smaller and users with average-size thumbs will mis-tap, especially women with longer nails or anyone walking while using the phone.

Before.menu-btn { font-size:24px; padding:0; }
actual: ~24×24px
After.menu-btn { min-width:44px; min-height:44px; padding:8px; }

5. GSAP hero parallax drained battery on mobile

Important

The hero background parallax scrub animation runs every scroll frame. On desktop with a discrete GPU that's fine. On a 4-year-old Android, it can drop frame rate to 20fps and visibly warm the phone after 30 seconds. Parallax adds zero conversion value on a 6-inch screen — there's barely any vertical distance for the parallax to register.

Before.hero-bg always animated regardless of viewport
After@media(max-width:880px){ .hero-bg { transform:none !important } }

6. Floating CTA could cover footer links

Important

The fixed-position "(703) 399-4394 / Quote" buttons sit at the bottom-right. On mobile, when you scroll to the bottom of the page, the CTA can overlap footer links and copyright text. Users couldn't tap "Privacy" or "Terms" if they wanted to.

Beforebody has no padding-bottom; CTA overlaps footer
Afterbody { padding-bottom: 100px } on mobile — CTA never covers content

7. No SMS option for nightlife crowd

Recommended

Party-bus customers are often coordinating with 8–15 friends. Many would rather text than call. Adding sms: links alongside tel: links gives them a low-pressure way to ask "is the 30-passenger available this Saturday?" — and a text thread is way easier for me to follow up on than voicemail.

Recommendation: Add a "Text us" button next to "Call now" using href="sms:+17033994394?body=Hi, I need a party bus for..."

8. Heading sizes too large on small phones

Polish

The H1 used clamp(48px, 8vw, 96px). On an iPhone SE (375px wide), that's ~48px — fine, but the line-height wrapped awkwardly. Tightened to clamp(36px, 9vw, 56px) on ≤680px and 34px on ≤380px for clean two-line headlines.

9. Default blue tap highlight broke brand

Polish

When you tap a link on iOS, the OS shows a quick blue flash. Looks cheap on a premium cream/gold brand. Overrode with -webkit-tap-highlight-color: rgba(201,169,97,0.25) so the flash matches the gold accent.

10. No focus-visible style for keyboard users

A11y

Mobile users using external keyboards (or accessibility tools) couldn't see what was focused. Added a clean gold outline via :focus-visible that only shows for keyboard nav, not mouse clicks.

03 · Fixes Applied

What's now deployed to every page

A single mobile-fixes CSS block was injected into all 92 HTML files. Every page now passes the same mobile bar.

Tap & Touch

  • Hamburger button: 44×44px minimum tap target
  • All .btn elements: 44px min-height on touch devices
  • Nav + footer links: 8px vertical padding for easier tapping
  • Mobile-nav drawer links: 14px padding, 44px min height
  • Tap highlight: gold instead of default blue

Forms

  • All inputs: 16px font-size — kills iOS zoom-on-tap
  • Input text color: warm dark #2d2418 (readable on cream)
  • Placeholder color: #8a7755 (visible but secondary)
  • Removed default iOS rounded-corner styling
  • Label color override so "Date, Time, Passengers" labels show

Layout

  • Tables auto-scroll horizontally on screens ≤880px
  • All multi-column grids collapse to 1 column on phones
  • Stat rows collapse to 2 columns (not 1) for visual balance
  • Hero/section padding tightened on small screens
  • body gets padding-bottom:100px so float-CTA doesn't cover content

Performance

  • Hero parallax disabled on screens ≤880px
  • Existing prefers-reduced-motion guard preserved
  • Images already had max-width:100%; height:auto
  • Existing loading="lazy" on most images
  • GSAP entry animations still run (cheap, one-shot)

Typography

  • Body base: 16px (matches form input rule)
  • H1 mobile: clamp(36px, 9vw, 56px)
  • H2 mobile: clamp(30px, 7vw, 48px)
  • iPhone SE (≤380px): H1 → 34px, H2 → 28px
  • Line-height 1.05–1.1 for tight, magazine-style headlines

Accessibility

  • :focus-visible outline for keyboard users (gold, 2px)
  • All hover states converted to (hover:none)-safe
  • Touch target sizing matches WCAG 2.1 AA
  • Color contrast: text on cream is 7:1+ (passes AAA)
  • Mobile menu uses aria-label friendly markup

05 · How to test

Verify the fixes yourself

After redeploying, here's exactly how to spot-check the mobile experience.

Open partybusrus.com on your phone

Real device, not just Chrome DevTools. Use an iPhone if you have one — that's the hardest browser to please.

Tap the hamburger menu

Should open immediately. The mobile drawer should cover the whole screen with cream background and large gold links.

Scroll the home page top to bottom

No horizontal scrolling anywhere. Everything fits in the viewport. Cards stack neatly. Photos look sharp.

Tap into a form field on Quote page

The page should NOT zoom in. If it zooms, the 16px fix didn't load — check that you redeployed.

Try the comparison table on the home page

You should be able to swipe it left/right inside its container without the whole page moving.

Tap the (703) 399-4394 button

Phone app should pre-fill the number ready to call. Test on real device — emulators don't simulate this perfectly.

Visit a fleet detail page

Bus photos should be full-width, specs grid stacks vertically, "Get a Quote" button at bottom is easy to thumb.

Open a city page (e.g. Tysons, VA)

Headline scales nicely, content reads cleanly, internal links all tap-friendly.

Test in landscape orientation

Rotate phone sideways. Layout should still work — float CTA on side, content fills width.

Try a slow connection (4G/3G throttle in Chrome)

First contentful paint should still happen within 2.5s. Hero image lazy-loads.
