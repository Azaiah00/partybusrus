1

Install Node.js (one-time)

Vercel CLI runs on Node.js. If you've never installed it:

  1. Go to nodejs.org/en/download
  2. Download the LTS Windows Installer (.msi)
  3. Run the installer — accept all defaults, click Next through everything
  4. Restart your terminal (PowerShell or Command Prompt) after install

Verify it worked — open PowerShell and run:

node --version
npm --version

You should see version numbers like v20.x.x and 10.x.x. If you do, you're set.

2

Install Vercel CLI

In PowerShell, run:

npm install -g vercel

Wait for it to finish (~30 seconds). Then verify:

vercel --version
3

Log into Vercel

vercel login

It'll ask how you want to log in — pick the same method you used to create your Hobby account (probably GitHub or email). A browser window opens, you click "Authorize", come back to the terminal, done.

4

Navigate to the site folder

Your site is at:

cd "C:\Users\LATITUDE-7400\Documents\Claude\Projects\Party Bus R Us\site"

Copy-paste that exactly into PowerShell.

5

Deploy

vercel

First time it'll ask:

  • Set up and deploy? → Yes
  • Which scope? → Your account (usually the only option)
  • Link to existing project? → No
  • Project name? → partybusrus (or whatever you want)
  • Directory? → press Enter (default ./)
  • Modify settings? → No

It uploads the files (~30 seconds) and gives you a URL like https://partybusrus-abc123.vercel.app. That's your preview URL.

6

Push to production

The first vercel command deploys to a preview URL. To make it live on the main Vercel URL, run:

vercel --prod

Done. The site is live at https://partybusrus.vercel.app (or whatever project name you used).

7

Connect your custom domain (partybusrus.com)

  1. Go to vercel.com/dashboard
  2. Click your partybusrus project
  3. Top tabs → Settings → Domains
  4. Add domain → enter partybusrus.com and www.partybusrus.com
  5. Vercel will show you DNS records to add at GoDaddy:
    • For partybusrus.com: A record pointing to 76.76.21.21
    • For www.partybusrus.com: CNAME pointing to cname.vercel-dns.com
  6. At GoDaddy: DNS Management → find your partybusrus.com domain → add the records Vercel gave you. Remove the OLD Netlify A/CNAME records.
  7. Wait 5-60 minutes for DNS propagation
  8. SSL certificate auto-provisions. Site is live at https://partybusrus.com

Privacy upgrade (do this after first successful submission)

Right now the form action contains your email in the page source: action="https://formsubmit.co/hello@realestateadvancement.com". Spammers crawl HTML for emails. To hide it:

🔒

Switch to hashed URL

  1. After your first form submission is delivered, check the email from FormSubmit
  2. It'll include a "Random Endpoint" URL like https://formsubmit.co/el/abc123XYZ
  3. Open site/quote.html
  4. Find: action="https://formsubmit.co/hello@realestateadvancement.com"
  5. Replace with: action="https://formsubmit.co/el/abc123XYZ" (your hashed URL)
  6. Redeploy: vercel --prod

Your email is now hidden from page source but submissions still arrive at the same inbox.

Other notes

📝
Build minutes

Vercel Hobby gives 100GB-hours of compute and 100GB of bandwidth per month free. For a static site like this, you'll use maybe 1% of that. Build minutes aren't really a concept on Vercel the way they are on Netlify.

🔄
If you ever go back to Netlify

The original _redirects file is still in the folder, untouched. To go back: revert quote.html form attributes to data-netlify="true", delete vercel.json, drag onto Netlify. Both setups can coexist in the same folder.

⚠️
DNS gotcha

While DNS propagates (5-60 min), some users might see the old Netlify site and some the new Vercel site. To force-update, hard-refresh (Ctrl+Shift+R) and clear browser cache. Use dnschecker.org to confirm worldwide DNS update.
